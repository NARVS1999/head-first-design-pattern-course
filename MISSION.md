# Mission: Head First Design Patterns

## Why
Matutunan ang mga proven design patterns para makabuo ng extensible, maintainable, at reusable na OO software. Ang goal ay maging malinaw ang pag-iisip sa pagdedesign ng systems — hindi lang pag-coode, kundi pag-iisip ng *paano* i-structure ang code para hindi mag-implode kapag lumaki ang project.

## Success looks like
- Mailarawan at ma-apply ang bawat major design pattern sa real-world scenarios
- Ma-identify kung aling pattern ang angkop sa isang design problem
- Ma-implement ang patterns nang tama sa Java (at ma-extend sa ibang languages)
- Ma-explain ang trade-offs ng bawat pattern sa iba pang developers
- Ma-build ng isang Smart Home System na gumagamit ng multiple patterns

## Constraints
- Sundin ang Head First Design Patterns (2nd Edition) bilang primary source
- Taglish (Tagalog-English) ang medium ng instruction
- Interactive lessons na may hands-on exercises
- Tokyo newspaper theme para sa visual design

## Out of scope
- Advanced enterprise patterns (EJB, J2EE specific)
- Performance optimization techniques
- Testing frameworks (JUnit, etc.)
- Database design patterns
