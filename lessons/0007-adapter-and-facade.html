Adapter & Facade

Pagkakabagay at Simpleng Interface — Being Adaptive

Lesson Goal Matutunan ang Adapter (baguhin ang interface para maging compatible) at Facade (magbigay ng simpleng interface para sa komplikadong subsystem).

Core Concepts

PatternPurposeAnalogy
AdapterBaguhin ang interface ng isang klase para maging compatible sa iba pang interface.Power plug converter — isang plug, maraming outlets.
FacadeMagbigay ng simpleng interface para sa isang komplikadong subsystem.Hotel concierge — iisang tao, lahat ng services.

Design Principle

"Principle of Least Knowledge — talk only to your immediate friends."

Sa Taglish: Makipag-usap lang sa mga direktang kaibigan mong objects, huwag sa kaibigan ng kaibigan. Ito ang Law of Demeter.

Sample Code (Adapter) 

// Turkey has a different interface than Duck
interface Turkey {
    void gobble();
    void fly();
}

// Adapter: makes Turkey quack like a Duck
class TurkeyAdapter implements Duck {
    Turkey turkey;
    public void quack() { turkey.gobble(); }
    public void fly() {
        // turkeys can fly, but only short distances
        for(int i=0; i<5; i++) turkey.fly();
    }
}

Gumawa ng Facade para sa Home Theater system (Projector, Amplifier, Player). Yung watchMovie() ay mag-o-on sa lahat ng devices at mag-s-setup ng tamang configuration.

Summary

  • Adapter: Nagbabago ng interface para maging compatible — parang power plug converter.
  • Facade: Nagbibigay ng simpleng interface sa komplikadong subsystem — hindi nag-aalis ng functionality.
  • Principle of Least Knowledge: Makipag-usap lang sa mga immediate friends mong objects.
