Leftover Patterns

Builder Pattern

Ginagamit para sa pagbuo ng complex objects step by step. Pagkatapos ng construction, ang product ay fully formed na.

// Builder pattern — step by step construction
Pizza pizza = new Pizza.Builder("Large")
    .cheese(true)
    .pepperoni(true)
    .mushrooms(false)
    .build();

Chain of Responsibility

Nagpasa ng request sa isang chain ng handlers. Bawat handler ay nagde-decide kung siya ang magpo-process o magpa-pass sa susunod.

// Chain of Responsibility
support.setNext(l3Support).setNext(l2Support).setNext(l1Support);
support.handle(request); // walks through the chain

Design Principles Recap

  1. Encapsulate what varies.
  2. Favor composition over inheritance.
  3. Program to interfaces, not implementations.
  4. Strive for loosely coupled designs.
  5. Classes should be open for extension, closed for modification.
  6. Depend on abstractions. Do not depend on concrete classes.
  7. Talk only to your immediate friends.
  8. Don't call us, we'll call you (Hollywood Principle).
  9. A class should have only one reason to change.
