Proxy Pattern

Kinokontrol ang Access — Controlling Object Access

Lesson Goal Gumawa ng surrogate o placeholder para sa ibang object para kontrolin ang access, at matutunan ang Remote, Virtual, at Protection Proxy.

Core Concepts

Proxy TypePurposeExample
Remote ProxyRepresents an object in a different address space (e.g., another JVM).Java RMI stubs
Virtual ProxyDelays creation of expensive objects until needed (lazy loading).ImageProxy na naglo-load ng image on demand
Protection ProxyControls access based on permissions.File proxy na humihingi ng password

Fig 1. Proxy Pattern — ImageProxy loads RealImage on demand

Sample Code (Virtual Proxy) 

class ImageProxy implements Icon {
    ImageIcon realImage;
    String imageURL;

    public void paintIcon(...) {
        if (realImage == null) {
            realImage = new ImageIcon(imageURL); // lazy loading!
        }
        realImage.paintIcon(...);
    }
}

Summary

  • Ang Proxy pattern ay nagbibigay ng surrogate o placeholder para sa ibang object.
  • Remote Proxy: Represents objects in different address spaces.
  • Virtual Proxy: Delays creation of expensive objects (lazy loading).
  • Protection Proxy: Controls access based on permissions.
