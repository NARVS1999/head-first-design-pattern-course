Phase 3 — Singleton at Command

Singleton Pattern

One and Only One — One of a Kind Objects

Lesson Goal Tiyaking iisa lang ang instance ng isang klase sa buong application, at matutunan ang iba't ibang implementation variants.

Core Concepts

ImplementationDescriptionPros/Cons
EagerInstance agad kapag nag-load ang classSimple, thread-safe. Pero hindi lazy.
LazyInstance lang kapag kailanganEfficient. Pero hindi thread-safe.
Double-Checked LockingLazy + synchronized block + volatileLazy at thread-safe. Pero complex.

Fig 1. Singleton Pattern — private constructor, static getInstance(), thread-safe

Sample Code (Thread-Safe) 

public class Singleton {
    private static volatile Singleton uniqueInstance;
    private Singleton() {} // private constructor!

    public static Singleton getInstance() {
        if (uniqueInstance == null) {
            synchronized (Singleton.class) {
                if (uniqueInstance == null) {
                    uniqueInstance = new Singleton();
                }
            }
        }
        return uniqueInstance;
    }
}
Warning Ang Singleton pattern ay madalas misused. Gamitin lang kapag talagang kailangan ng iisang instance — hal. database connection pool, configuration manager. Huwag gawing Singleton ang lahat!

Summary

  • Ang Singleton ay nagsisiguro na iisa lang ang instance ng isang klase.
  • May tatlong variants: Eager, Lazy, at Double-Checked Locking.
  • Private constructor at static getInstance() method.
  • Mag-ingat sa overuse — hindi lahat kailangan ng Singleton!
