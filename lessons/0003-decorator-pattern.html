Phase 2 — Decorator at Factory

Decorator Pattern

Dinadagdagan ng Powers — Decorating Objects

Lesson Goal Matutunan kung paano magdagdag ng behavior sa objects nang hindi gumagamit ng inheritance, at ma-appreciate ang Open/Closed Principle.

Core Concepts

Ang Decorator Pattern ay nagbibigay-daan sa pagdagdag ng behavior sa objects nang hindi binabago ang original object. Binebalot (wrap) mo ang isang object sa loob ng isa pang object na may parehong interface.

Key Insight Ang mga decorators ay maaaring mag-stack — pwede kang mag-wrap ng Mocha sa loob ng Whip sa loob ng DarkRoast. Ang total price ay na-accumulate sa bawat level!

Design Principle

"Classes should be open for extension, but closed for modification."

Sa Taglish: Pwedeng dagdagan ang features ng isang klase, pero hindi mo kailangang baguhin ang existing code nito. Ito ang Open/Closed Principle.

Fig 1. Decorator Pattern — CondimentDecorators wrap Beverage objects

Sample Code 

// Build a beverage with decorators
Beverage beverage = new DarkRoast();
beverage = new Mocha(beverage);      // wrap with Mocha
beverage = new Whip(beverage);      // wrap with Whip
System.out.println(beverage.cost());
// DarkRoast: $0.99 + Mocha: $0.20 + Whip: $0.15 = $1.34

Summary

  • Ang Decorator pattern ay nagdadagdag ng behavior sa objects sa pamamagitan ng composition (wrapping), hindi inheritance.
  • Ang mga decorators ay naka-serialize — pwedeng mag-stack ng maraming decorators.
  • Open/Closed Principle: Ang klase ay open for extension pero closed for modification.
  • Kapag maraming variations ng behavior, mas mainam ang decorators kaysa sa maraming subclasses.
