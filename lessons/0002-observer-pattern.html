The Observer Pattern

Huwag Mong Miss ang Update! — Keeping your Objects in the Know

Lesson Goal Maunawaan ang one-to-many dependency sa pagitan ng objects at ma-implement ang sariling Observer pattern gamit ang interfaces.

Core Concepts

Ang Observer Pattern ay nagbibigay-daan sa isang object (ang Subject) na automatic na mag-notify sa maraming dependent objects (ang mga Observers) kapag nagbago ang state nito.

RoleDescriptionExample
Subject (Observable)Ang source ng data/events. Nag-iimbak ng list ng observers at nagno-notify sa kanila.WeatherData
ObserverMga nag-a-abang ng update mula sa Subject. Bawat isa ay may update() method.PhoneDisplay, TabletDisplay
Loose Coupling Alam lang ng Subject na may observers, pero hindi niya alam ang konkretong klase nila. Ito ang tawag na loose coupling — pwedeng mag-add/remove ng observers nang hindi na-aapektuhan ang Subject.

Design Principle

"Strive for loosely coupled designs between objects that interact."

Sa Taglish: Gawing maluwag ang koneksyon ng mga nag-uusap na objects — para pwede mong baguhin ang isa nang hindi masyadong apektado ang isa.

Observer Pattern Structure

Fig 1. Observer Pattern — Subject notifies multiple Observer implementations

Simpleng Code 

interface Subject {
    void registerObserver(Observer o);
    void removeObserver(Observer o);
    void notifyObservers();
}

interface Observer {
    void update(float temp, float humidity);
}

class WeatherData implements Subject {
    private List<Observer> observers = new ArrayList<>();
    private float temperature;
    private float humidity;

    public void registerObserver(Observer o) {
        observers.add(o);
    }

    public void notifyObservers() {
        for (Observer observer : observers) {
            observer.update(temperature, humidity);
        }
    }

    public void setMeasurements(float t, float h) {
        this.temperature = t;
        this.humidity = h;
        notifyObservers();
    }
}

Summary

  • Ang Observer pattern ay nagbibigay ng one-to-many dependency: ang Subject ay nagno-notify sa maraming Observers.
  • Loose coupling — hindi alam ng Subject ang konkretong klase ng observers nito.
  • Ang Subject ay may registerObserver(), removeObserver(), at notifyObservers().
  • Ang Observer ay may update() method na tinatawag kapag nagbago ang state ng Subject.
