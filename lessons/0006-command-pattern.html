Phase 3 — Singleton at Command

Command Pattern

I-encapsulate ang mga Request — Encapsulating Invocation

Lesson Goal Gawing object ang isang request o aksyon, at i-decouple ang sender ng request mula sa receiver.

Core Concepts

RoleDescriptionExample
CommandInterface na may execute() method. Nag-encapsulate ng isang request bilang object.LightOnCommand, TVOffCommand
InvokerNag-iimbak at nagta-trigger ng commands. Hindi alam kung ano ang gagawin ng command.RemoteControl
ReceiverAng talagang gagawa ng trabaho. Alam kung paano i-perform ang action.Light, TV, Stereo

Fig 1. Command Pattern — Invoker triggers Commands that act on Receivers

Sample Code 

interface Command {
    void execute();
    void undo();
}

class LightOnCommand implements Command {
    Light light;
    public void execute() { light.on(); }
    public void undo() { light.off(); }
}

class RemoteControl {
    Command[] onSlots;
    Command[] offSlots;

    public void pressOn(int slot) {
        onSlots[slot].execute();
    }
    public void pressUndo() {
        lastCommand.undo();
    }
}

Summary

  • Ang Command pattern ay nag-encapsulate ng request bilang isang object.
  • May tatlong roles: Command (interface), Invoker (nag-trigger), Receiver (nag-perform).
  • Pwedeng mag-implement ng undo functionality.
  • Nag-de-couple ang sender ng request mula sa receiver.
