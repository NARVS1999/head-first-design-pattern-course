Compound Patterns

Patterns na Nagtutulungan — Patterns of Patterns

Lesson Goal Makita kung paano pinagsasama-sama ang maraming pattern sa iisang application — ang MVC (Model-View-Controller) bilang compound pattern.

Core Concepts: MVC

MVC ComponentPattern UsedDescription
ModelObserverAng data at business logic. Nag-i-notify sa views kapag nagbago ang data.
ViewObserverAng UI. Nag-o-observe sa model para mag-update.
ControllerStrategyNag-handle ng user input at nag-uumap sa model/view.

Fig 1. MVC Architecture — Model (Observer), View (Observer), Controller (Strategy)

Sample Code (DJ Beat MVC) 

// Model — Observable
class BeatModel {
    int bpm;
    List<BeatObserver> observers = new ArrayList<>();
    void setBPM(int bpm) {
        this.bpm = bpm;
        notifyObservers();
    }
}

// View — Observer
interface BeatObserver {
    void updateBeat();
}

// Controller — Strategy
class BeatController {
    BeatModel model;
    void increaseBPM() { model.setBPM(model.getBPM() + 1); }
    void decreaseBPM() { model.setBPM(model.getBPM() - 1); }
}

Summary

  • Ang MVC ay isang compound pattern na nagsasama ng Observer, Strategy, at Composite.
  • Model: Observer — nag-i-notify sa views.
  • View: Observer — nag-o-observe sa model.
  • Controller: Strategy — nag-handle ng user input.
  • Ang mga patterns ay nagtutulungan sa loob ng isang application.
