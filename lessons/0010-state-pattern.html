State Pattern

Nagbabago ang Ugali — The State of Things

Lesson Goal Payagan ang object na baguhin ang behavior kapag nagbago ang internal state, at maiwasan ang maraming if-else o switch.

Core Concepts

ComponentDescriptionExample
ContextAng object na nagdi-delegate sa current state object. Nagbibigay ng reference sa current state.GumballMachine
State InterfaceDefines ang mga actions na pwedeng gawin sa bawat state.insertQuarter(), ejectQuarter()
Concrete StatesBawat state ay isang klase na nagpapatupad ng behavior para sa state na iyon.NoQuarterState, HasQuarterState

Fig 1. State Machine — GumballMachine transitions between states

Sample Code 

interface State {
    void insertQuarter();
    void ejectQuarter();
    void turnCrank();
    void dispense();
}

class HasQuarterState implements State {
    GumballMachine machine;
    public void insertQuarter() { System.out.println("Already have quarter"); }
    public void turnCrank() {
        System.out.println("Turning...");
        machine.setState(machine.getSoldState());
    }
}

Summary

  • Ang State pattern ay nagbibigay-daan sa object na baguhin ang behavior depende sa internal state.
  • Bawat state ay isang klase na nag-iimplement ng iisang interface.
  • Ang Context ay nagdi-delegate sa current state object.
  • Nag-iwas sa maraming if-else o switch — mas malinis at maintainable.
