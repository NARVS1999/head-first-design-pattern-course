Phase 2 — Decorator at Factory

Factory Patterns

Simple, Method, at Abstract — Baking with OO Goodness

Lesson Goal Maunawaan ang tatlong factory (Simple, Method, Abstract), at makita kung paano ini-encapsulate ang object creation.

Core Concepts

Factory TypeDescriptionKailan Gamitin
Simple FactoryIsang klase na may paraan para gumawa ng objects. Hindi technically isang pattern, kundi isang programming technique.Kapag may iisang klase na nagpoproduce ng iba't ibang objects depende sa parameter.
Factory MethodSubclasses ang nagde-decide kung anong object ang gagawin. Ang base class ay nagde-define ng algorithm.Kapag hindi mo alam kung anong exact na klase ang gagamitin hanggang sa runtime.
Abstract FactoryInterface para sa creating families of related objects nang hindi tinutukoy ang mga concrete classes.Kapag may multiple na may kaugnay na mga products na kailangan i-create together.

Design Principle

"Depend upon abstractions. Do not depend upon concrete classes."

Sa Taglish: Ang code mo dapat nakadepende sa interfaces o abstract classes, hindi sa mga konkretong implementation. Ito ang Dependency Inversion Principle.

Fig 1. Factory Method Pattern — PizzaStore delegates pizza creation to subclasses

Sample Code (Factory Method) 

abstract class PizzaStore {
    abstract Pizza createPizza(String type);

    public Pizza orderPizza(String type) {
        Pizza pizza = createPizza(type); // factory method!
        pizza.prepare();
        pizza.bake();
        pizza.cut();
        pizza.box();
        return pizza;
    }
}

class NYPizzaStore extends PizzaStore {
    Pizza createPizza(String type) {
        if (type.equals("cheese")) return new NYStyleCheesePizza();
        // ...
    }
}

Summary

  • Simple Factory: Isang klase na nagpoproduce ng objects — technique, hindi pattern.
  • Factory Method: Subclasses ang nagde-decide kung anong object ang gagawin.
  • Abstract Factory: Interface para sa creating families of related objects.
  • Dependency Inversion: Depend sa abstractions, hindi sa concrete classes.
