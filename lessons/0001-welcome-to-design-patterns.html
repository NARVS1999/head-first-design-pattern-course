Ano ang Design Pattern?

Ang design pattern ay isang tested na solusyon sa paulit-ulit na problema sa software design. Hindi ito code na pwedeng i-copy-paste lang — ito ay isang template o idea na pwedeng i-adapt sa iba't ibang sitwasyon.

Tandaan Lahat ng patterns ay may trade-offs — kailangan mong pumili ng tama depende sa sitwasyon. Walang pattern na "perfect" para sa lahat ng problems.

Isipin mo ito parang recipe sa pagluluto: hindi mo basta binabato ang ingredients — may tamang paraan, tamang顺序 (order), at tamang resulta. Ganoon din sa design patterns.

Unang Design Principle

"Identify the aspects of your application that vary and separate them from what stays the same."

Sa Taglish: Hanapin mo yung mga bahagi ng code na madalas magbago, at ihiwalay mo sila para hindi maapektuhan ang stable na code.

Ito ang pinaka-fundamental na principle sa buong course. Kapag na-master mo ito, magiging mas malinaw ang pag-iisip mo sa pagdedesign.

Sample Idea: Encapsulating What Varies 

// Instead of hardcoding behavior, use an interface
interface QuackBehavior {
    void quack();
}

class Duck {
    QuackBehavior quackBehavior; // varies!

    void performQuack() {
        quackBehavior.quack(); // delegates to behavior
    }
}

// Different implementations
class Squeak implements QuackBehavior {
    void quack() { System.out.println("Squeak!"); }
}

class MuteQuack implements QuackBehavior {
    void quack() { System.out.println("<< silence >>"); }
}

Summary

  • Ang design pattern ay isang tested na solusyon sa paulit-ulit na design problem — hindi lang code, kundi isang idea.
  • Ang unang design principle: Identify what varies and separate it from what stays the same.
  • Ang OO basics (inheritance, encapsulation, polymorphism, abstraction) ay ang foundation ng lahat ng patterns.
  • Lahat ng patterns ay may trade-offs — pumili ng tama depende sa sitwasyon.
